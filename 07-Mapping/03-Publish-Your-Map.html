Publish Your Map on the Web¬∂

Here are two (free!) options for publishing your Folium map on the web.

Netlify¬∂

One of the easiest ways to publish your map is to drag and drop your map into Netlify.

All you need to do is:

  1. Create a folder for your web map (which you can name whatever you want, e.g. ‚Äúmy-web-map‚Äù)

  2. Change the name of your map HTML file to ‚Äúindex.html‚Äù

  3. Move ‚Äúindex.html‚Äù inside your web map folder

  4. Then drag and drop this folder into Netlify

Netlify will create a live site with your map (at a random URL), which will stay online for 24 hours. To make this site permanent and/or change the site URL, you can sign up for a free Netlify account. To change the site‚Äôs name, go to Settings -> Site Details -> ‚ÄúChange site name‚Äù

You can check out a sample map published with Netlify here: https://ithaca-map.netlify.app/

GitHub Pages¬∂

You can also publish your map as a simple website with GitHub Pages (as featured in the example above). Below I‚Äôm going to walk through the key points of the GitHub Pages tutorial for creating a ‚ÄúProject site.‚Äù

Though you don‚Äôt need to be a GitHub expert to make a GitHub Pages website, understanding the following premises will be helpful:

  • GitHub consists of ‚Äúrepositories‚Äù where you can store files and code for your projects

  • Git (the backbone of GitHub) is a version-control system that helps you keep track of the changes in your projects. Whereas a platform like Google Docs automatically keeps track of changes that you make in a document, Git needs you to record and track your own changes in a more explicit way. You will need to ‚Äúcommit‚Äù any changes that you make (e.g., adding your map file) and include a message that describes this change (e.g., ‚Äúadded a map file‚Äù).

For more information, refer back to our course materials Git & GitHub or see GitHub‚Äôs official guides.

Create a New Repository¬∂

If you don‚Äôt have one already, create a GitHub account. Then create a ‚ÄúNew repository‚Äù.

Note

Your free GitHub Pages website will be hosted at a URL that combines your GitHub user name and your repository name, like so https://user-name.github.io/repo-name/ So be sure to choose your names wisely!

Provide a name for your repository and then click ‚ÄúCreate Repository.‚Äù

Choose a GitHub Pages Theme¬∂

Choose a theme/style/layout for your website. Go to Settings -> GitHub Pages -> ‚ÄúChoose a theme‚Äù. For this tutorial, I‚Äôm choosing the ‚ÄúMinimal‚Äù theme.

Once you‚Äôve selected a theme, you can delete the text that‚Äôs included in the default ‚Äúindex.md‚Äù file. Whatever you put in this Markdown (.md) file will be featured on your web site‚Äôs home page. For now, you can put something simple in the file, add a ‚Äúcommit‚Äù message (such as ‚Äúadded website content‚Äù), and then ‚ÄúCommit changes‚Äù (you can ‚ÄúCommit directly to the master branch‚Äù).

Note

Markdown is a special markup language, which allows you to format and style text in a very simple way. For example, a hash mark # will format your text as a header and a pair of asterisks ** will format your text in italics.

Upload Map and Logo¬∂

To upload your map HTML file and (optionally) an image for a web site logo to your GitHub repository, click ‚ÄúUpload files.‚Äù

Then drag and drop your map HTML file and a logo image, type a commit message, and ‚ÄúCommit changes‚Äù.
